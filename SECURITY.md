# Security Policy

## Supported Versions

Voxel Sandbox Engine is currently in a **pre-alpha** stage of development.

At this time:
- No official releases are published
- APIs and internal structures may change without notice
- Security updates are provided on a best-effort basis

Only the `main` branch is actively maintained.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public GitHub issue**.

Instead, report it privately using one of the following methods:

- E-mail the repository owner or maintainer
- Use GitHubâ€™s private security advisory feature (if available)

Please include:
- A clear description of the issue
- Steps to reproduce (if applicable)
- Potential impact
- Any relevant logs or proof of concept

---

## Scope of Security Issues

Security reports may include, but are not limited to:
- Vulnerabilities leading to crashes or denial of service
- Data corruption or unintended data exposure
- Unsafe handling of external input or assets
- Misuse of database connections or credentials
- Unsafe native library loading

---

## Disclosure Policy

Security issues will be investigated as quickly as possible.

If a vulnerability is confirmed:
- A fix will be prepared privately
- A public disclosure may follow after a fix is available
- Credit may be given to the reporter, if desired

---

## Final Notes

This project is experimental and evolves incrementally.
Responsible disclosure helps keep the project safe for everyone.

Thank you for helping improve the security of Voxel Sandbox Engine.
